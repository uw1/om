# why not
Lisp要插件，没插件。要代码没代码？要资料没资料？如果想要高效编写，就必须不仅仅是什么代码都要自己写。而且还要自己写插件，自己写资料。投入的时间几十年都回不了本。
如果能够聚集到足够多的人，那么我为什么还要去写Lisp插件？而不是重新去发明Lisp呢。所以注定了是一个对个人不值得，对集体也不值得使用的语言

# bad
市面上所有Lisp语言只是把各种符号统一成了括号，但是却并没有减少符号。并没有死语法更简单，只是更单一。因此在语法上其实并没有优势，只是各有优劣。
	eg. arr[1] → (. arr 1) , a = b → (= a b)
	并且在没有专用编辑器配合的情况下，根本发挥不出优势。
	所以说目前为止完全不值的使用。
把没有结构化的了，代码也强行结构化，就显得非常蠢。
